# OpenAgent Highlights

OpenAgent turns Obsidian Canvas into a local workspace for working with Codex.

## What Makes It Useful

- **Work starts where thinking happens**
  You can select nodes directly on an Obsidian Canvas and turn them into Codex tasks.

- **Canvas becomes a live workspace**
  Nodes are not just notes anymore. They can show running state, completed state, and connected results.

- **Results come back to the canvas**
  Assistant replies are written back as result nodes, linked to the original source node.

- **Context stays visible**
  The prompt, supporting notes, and final answer stay together in the same visual space.

- **Each node can become a thread**
  A canvas can hold many focused conversations: bugs, ideas, research notes, plans, and follow-ups.

- **Follow-ups feel natural**
  You can continue from another node instead of copying context into a separate chat window.

- **Local-first by design**
  Obsidian is the interface, the daemon runs locally, and the repo stays on your machine.

- **Good for real project work**
  It fits workflows like code review, architecture planning, debugging, documentation, onboarding, and research.

- **Less context switching**
  You do not need to jump between notes, terminal, repo, and AI chat as much.

- **The project history becomes visual**
  The canvas turns into a map of questions, answers, decisions, and next actions.

## Simple Pitch

OpenAgent makes Obsidian Canvas feel less like a static planning board and more like a local operating surface for agentic coding.

You think in nodes.
Codex works from those nodes.
The result comes back to the graph.
